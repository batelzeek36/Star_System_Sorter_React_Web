# Documentation Organization

This folder contains all project documentation organized by purpose.

## Folder Structure

### `/project-history/`
**Completed milestone documentation** - Historical record of major achievements:
- Gate-line API generation
- Integration completions (gate-line, sources, hexagrams)
- Feature launches (GPT-4o narrative, scoring upgrades)
- Data pipeline completions (Legge sanitization, hexagram overlays)

### `/status-reports/`
**Point-in-time status snapshots** - Verification and audit reports:
- App status reality checks
- Data completeness reports
- Integration status updates
- Verification reports

### `/guides/`
**How-to and quick-start guides** - Active reference for common tasks:
- Quick start guides (gate-line scoring, research sources)
- Quick reference cards (scoring v4.3)
- Checklists (narrative cache MVP)

### `/specifications/`
**Technical specifications** - Canonical format and design docs:
- Batch file specification
- Shadow-as-distortion framework
- Sparsify settings (gentle/sharpen)

### `/strategy/`
**Strategy and planning documents** - Vision and roadmap:
- Fix strategies (scoring)
- Upgrade plans (narrative caching)
- Roadmaps (narrative production)
- Philosophy docs (why rigor matters)
- Next steps planning

### `/working/`
**Scratch and temporary docs** - Work-in-progress:
- Plans and prompts
- Task lists
- Reference materials
- Ready-to-test notes

## Finding What You Need

- **"How do I...?"** → `/guides/`
- **"What's the format for...?"** → `/specifications/`
- **"When did we complete...?"** → `/project-history/`
- **"What's the current status of...?"** → `/status-reports/`
- **"What's the plan for...?"** → `/strategy/`
- **"Where's that scratch doc about...?"** → `/working/`
